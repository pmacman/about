# Project Portfolio

This account contains a **curated collection of projects** developed across different stages of my career.

The repositories here are intentionally diverse in age, tooling, and scope. Rather than presenting only the newest stack, this portfolio emphasizes **technical progression, architectural thinking, and long-term learning**.

Not every project represents how I would build something *today* — but every project represents something I've built, learned from, and understood deeply.

## Projects

- [vite-react-router-starter](vite-react-router-starter)
    - This repository is a modern React + TypeScript starter project built on Vite and React Router. It emphasizes clear project structure, explicit routing, and predictable configuration.
- [react-hook-spreadsheet](react-hook-spreadsheet)
    - An example React project demonstrating how to generate Excel spreadsheets using ExcelJS via a custom, reusable React hook.
- [react-mui-components](react-mui-components)
    - A small React project showcasing custom, reusable Material UI (MUI) components built with flexibility and real-world usage in mind.
- [gatsby-contentful-demo](gatsby-contentful-demo)
    - This repository is a streamlined demonstration project built to document and preserve a working setup for a Gatsby + Contentful website.
- [mosaic-design-tool](mosaic-design-tool)
    - The Mosaic Designer mobile app is a design tool for creating custom mosaic tile patterns. An example would be a decorative backsplash for a kitchen or bathroom. Typical users of this application would include customers, architects, showroom consultants, and outside sales representatives.
- [bootstrap-custom-theme](bootstrap-custom-theme)
   - A customizable, versioned Bootstrap theme system built with Sass / Less and Webpack, designed to support multiple major Bootstrap versions from a shared design foundation.
- [webpack-config-reference](webpack-config-reference)
    - This repository is a legacy reference project demonstrating how to manually configure Webpack 4 and Webpack 5 without relying on framework abstractions.
- [estore-java-demo](estore-java-demo)
    - E-store is a legacy Java-based e-commerce application developed to demonstrate service-oriented architecture (SOA), layered backend design, and full-stack web application concepts.

## How to Read This Portfolio

Projects generally fall into one of three categories:

### 🟢 Modern / Active

Projects using current tools and patterns. These reflect how I typically work today.

Examples:

- React + TypeScript
- Modern component design
- Hooks and reusable abstractions
- Vite / modern build tooling

### 🟡 Demo / Reference

Smaller, focused projects created to:

- Capture a working setup
- Document a specific integration
- Serve as a personal reference or teaching aid

These projects are intentionally minimal and scoped.

Examples:

- Static site generation demos
- CMS integrations
- Configuration-focused projects

### 🔴 Historical / Foundational

Older projects that reflect:

- Earlier tooling ecosystems
- Architectural patterns common at the time
- End-to-end system design experience

These are preserved for context and learning, not as recommendations for new greenfield work.

Examples:

- Manual Webpack configurations
- Hybrid mobile applications
- Server-rendered MVC applications

## Why Keep Older Projects?

Technology changes quickly. Fundamentals do not.

These repositories demonstrate:

- Comfort working **below abstractions**
- Understanding of layered architectures
- Ability to adapt across generations of tooling
- Experience maintaining and reasoning about legacy systems

Modern frameworks make many problems disappear — but understanding *why* they exist is still valuable.

## What You Won't Find Here

- Tutorial-only code copied from blogs
- Inflated demo apps pretending to be production systems
- Trend-chasing repos with no long-term learning value

Every project exists for a reason.

## Notes on Maintenance

Many repositories here are **not actively maintained**. That is intentional.

Each README clearly states:

- The project's purpose
- Its historical or technical context
- Whether it is meant for active use, reference, or documentation
